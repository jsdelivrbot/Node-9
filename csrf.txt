	Webstorage is accessible through Javascript on the same domain.
	Cookies with the HttpOnly flag aren't accessible through Javascript, these are immune to XSS.
	Cookies are stateless, when you are storing an JWT inside them.
	
	Da HTTP requests die an die Domain gebundenen Cookies automatisch mitsendet, ist es möglich, dass eine Seite HTTP requests für Fremde Domains ausführt und dazu die evtl. vorhandenen session Cookies mitsendet. 
	Javascript kann nur Cookies auslesen, welche sich auf derselben domain befinden, auf der das Script ausgeführt wird.
	Gegen CSRF: token mithilfe von javascript aus Cookie auslesen und in HTTP header setzen
